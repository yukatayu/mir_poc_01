- 2026-05-04 型システムの current reading:
  current first strong typing layer は `finite decidable index fragment`。
  authority hierarchy、security label hierarchy、capture、region、cost は built-in ではなく `user-defined finite theory`。
  current repo evidence で見えるのは、authority / label / capture / cost / lifetime / fallback / variance / contract-compatibility の first floor までであり、full typed calculus completion ではない。

- 2026-05-04 型で何を証明できるか:
  現行 evidence では、権限不足 declassification reject、label flow reject、capture escape reject、cost bound reject、fallback successor capability strengthening reject、mutable covariance reject、monotone degradation、contract / failure / observation safety の selected floor を machine-check できる。
  ただし全性質を 1 本の強い型システムで完結させる設計ではなく、residual obligation、model-check second line、Lean side proof line を分離している。

- 2026-05-04 型の強弱 / 層の付け外し:
  「完成した attachable type-layer 製品」があるわけではない。

- 2026-05-04 dependent type / Isabelle 感:
  README は `full dependent type theory` 未完を明記している。
  実感としては Isabelle 的な integrated proof language というより、Mir semantic core の横に Lean mechanization spine がある構図に近い。
  `samples/lean/foundations/` は small actual proof fragment、`samples/lean/clean-near-end/` は generated theorem stub であり、後者は full domain discharge completion を意味しない。

- 2026-05-04 I/O と HelloWorld:
  Mir core には privileged `stdin` / `stdout` / `stderr` が無い。
  standard I/O は core primitive ではなく、typed external effect / adapter boundary 側に残す current decision である。
  したがって典型的な stdout HelloWorld を「Mir core 単体の代表 sample」としては持っていない。

- 2026-05-04 Mir の計算能力:
  Mir に計算能力が無いわけではない。
  current evidence では `perform`、`require`、`ensure`、`try/fallback`、`atomic_cut`、`publish/observe/handoff/witness`、property evaluation が動く。
  ただし current positioning は「汎用 scripting 言語」より「effect / contract / causality / rollback / verification を前面に出した意味論コア」である。

- 2026-05-04 native binary / binary split:
  current repo では Mir source から final native binary を生成できるとはまだ言えない。
  `plan/23-compiler-backend-llvm-guardrail-roadmap.md` は LLVM/backend の guardrail までで、actual LLVM build、final backend choice、installed-binary packaging success criteria は fixed していない。
  `README.md` と `Documentation.md` も current projection/codegen evidence を final emitted executable family / final public emitted-program ABI と読まないよう明記している。

- 2026-05-04 binary split の current reading:
  現時点では「Mir が place ごとに最終バイナリへ分割される」とは主張しない。
  `Place` はまず execution locus の意味論であり、安定した binary partition unit として public に凍結されていない。
  分割方法、配置指定、deployment planner、cross-place equivalence checker は kept-later / unresolved に属する。
