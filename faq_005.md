## 9. `lambda-circle-box` や guarded / MDTT / MTT とは、いまどう結びつけているのか

current repo の読みでは、先行研究との結びつきは
**採用済みの最終基盤を宣言すること**
ではなく、
**どの理論がどの問題を支えられるかを切り分けること**
にある。

現時点の整理は次である。

| 理論 family | current repo での役割 | いま対応づけているもの | まだ対応づけていないもの |
|---|---|---|---|
| `lambda-circle-box` | partial basis candidate | `stage / later / always` の micro-core、time-ordered normalization | place / authority / witness / durability の multi-axis 全体 |
| guarded lambda-calculus | stronger adjacent candidate | later-style reasoning、productivity、stream / `emit` 系の考察 | authority / witness / scope の full decomposition |
| Modal Dependent Type Theory | stronger candidate family | richer modal dependent basis、Fitch-style line | current repo への concrete integration |
| Multimodal Dependent Type Theory | stronger candidate family | place / scope / authority / witness の multi-axis を載せる comparison family | final adoption |

簡潔に言うと、

- `lambda-circle-box` は **この言語全体の最終基盤** ではなく、**stage / later / always を説明する部分基盤**
- guarded / MDTT / MTT は **複数軸 modality が必要になったときの stronger candidate**

という扱いである。

したがって FAQ 上でも、

- `lambda-circle-box` を final adopted calculus と書かない
- `MTT` を adopted lead と書かない
- あくまで current comparison / current recommendation として説明する

のが current repo と整合する。
