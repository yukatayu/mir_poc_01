# 12 — Decision Register

This document records major architectural decisions and their current strength level.

| ID | Topic | Current decision | Level | Notes |
|---|---|---|---|---|
| D-001 | Overall architecture | Keep Mir, Mirrorea, PrismCascade, and Typed-Effect Wiring Platform separate but interoperable | L1 | Core architectural stance |
| D-002 | Core implementation language | Prefer Rust for core implementations | L2 | Recommendation, not a law |
| D-003 | Graph evolution | Prefer downstream / leaf-style patching as the default evolution rule | L0/L1 | Strong foundational constraint |
| D-004 | API evolution | Forbid silent shadowing; allow compatibility-preserving overlays only | L0/L1 | Central to safe evolution |
| D-005 | Failure handling | Keep failure explicit and structured | L0/L1 | Exact formalization still being refined |
| D-006 | Prism runtime placement | Keep PrismCascade runtime separate from Mir runtime | L1 | Integration should happen at explicit boundaries |
| D-007 | Legacy integration | Permit tunnel/proxy-based wrapping of legacy systems with explicit boundary semantics | L2 | Exact guarantees vary by case |
| D-008 | Coroutine direction | Structured, restricted coroutine model appears plausible; unrestricted coroutine model is not accepted | L2 | Outside Mir-0 and still requires further semantics work |
| D-009 | Typed-effects wiring platform | Treat as separate but closely related subsystem | L2 | Exact placement relative to Mirrorea still open |
| D-010 | Tooling workflow | Every substantive Codex task must write a new report | L1 | Process discipline |
| D-011 | Mir-0 boundary | Fix Mir-0 as the minimum kernel consisting of event DAG, `place`, a minimum effect request operation (written here as `perform`), effect / contract, minimum failure space, primitive fallback, local `try` / rollback, place-local `atomic_cut`, and linear resources; defer `barrier`, `durable_cut`, full fallback normalization, `emit`, coroutine semantics, and overlay alias details | L1 | Stabilizes Workstream A without deciding broader Mir or Mirrorea machinery |
| D-012 | Mir-0 の cut / 語彙境界 | `atomic_cut` は current `place` の rollback frontier だけを確定する Mir-0 の最小 finalizing cut とする。単一プロセス全体の同期、分散合意、永続化完了は Mir-0 に含めない。`durable_cut` と `barrier` は Mir-1 以降の cut vocabulary 側に送る。`perform` は当面、最小 effect request operation を説明するための記法であり、Mir-0 の規範的な表面構文は未決定とする | L1 | Mir-0 を広げず、cut と語彙の境界だけを狭く固定する |
| D-013 | durable_cut / barrier の責務分担 | `durable_cut` は Mir-1 側の cut vocabulary 候補とし、`barrier` は、Mir-1 に残すとしても commit-like primitive ではなく ordering primitive 候補として扱う。Mirrorea は later cut vocabulary の意味づけは行わず、その storage / distributed realization / route / audit 側だけを引き受ける | L1 | Mir-0 / Mir-1 / Mirrorea の cut 境界を明確にする |
| D-014 | durable_cut の最小 guarantee set | `durable_cut` の最小追加保証は、abstract persistence requirement を伴う durable commit guarantee である。これは successful な pre-cut prefix が local rollback / process restart / route rebinding の後に未確定へ戻らないことを要求する。具体的な storage / replication / consensus mechanism は Mir-1 では固定しない。distributed finalization は最小意味そのものではなく、Mirrorea が複数 `place` / 実ノード上で引き受ける operational realization 側に属する | L1 | Mir-1 の cut 語彙を狭く定義し、Mirrorea への責務委譲を明示する |
| D-015 | durable_cut の failure surface | durable guarantee を確立できなかった `durable_cut` は、successful な cut が成立していない失敗試行として扱う。既定分類は `Reject` とし、cut 後の外部化された obligations を明示的に巻き戻す必要がある場合だけ `Compensate` を使う。`Approximate` は contract が durability を弱めた代替結果を明示的に許す場合に限って検討し、新しい failure class は導入しない | L1 | failure lattice を再利用し、Mir-1 と Mirrorea の境界を維持する |
| D-016 | durable_cut の observation surface | successful `durable_cut` は、ある cut attempt / pre-cut prefix が persistence evidence に支持されたことを示す明示的 event が立ったときにだけ観測されたものとみなす。failed `durable_cut` は、同じ attempt に対して successful event が立つ前に failure outcome が明示されたときに観測される。audit は、cut attempt、対象 prefix、success / failure outcome、evidence の観測有無を説明できる trace を残す。evidence の具象形式は Mir-1 では固定せず、Mirrorea の実現責務に残す | L1 | 観測面の最小意味だけを Mir-1 に残し、具象表現は Mirrorea に送る |
| D-017 | durable_cut の cross-place scope rule | cross-place `durable_cut` は participating `place` ごとの local cut attempt を束ねる aggregate attempt を観測単位とし、cross-place の `prefix` は local prefixes の有限対応として扱う。single-place の success / failure event は member-local observation として保持しつつ、cross-place `durable_cut` 自体は scope rule に従う aggregate success / failure event で観測する。scope rule の具体 family と evidence 形式は Mir-1 では固定せず、Mirrorea は member observation の収集・相関と aggregate event / audit representation の実現を担う | L1 | single-place semantics を壊さずに cross-place 束ねだけを最小追加する |
| D-018 | durable_cut の scope rule family | Mir-1 が現時点で必須とする cross-place scope rule profile は `all_of` のみとする。`all_of` は aggregate attempt に含まれる participating `place` のすべてについて required local observations / persistence evidence を success 条件に数える conjunction profile である。`quorum-like` は将来拡張候補として残すが current Mir-1 には含めない。`implementation-defined` は Mir-1 の profile 語彙にはせず、Mirrorea は Mir-1 が意味づけた `all_of` などの profile について同じ aggregate success / failure 観測を保つ形で実現する裁量だけを持つ | L1 | profile family を増やしすぎず、Mir-1 の意味と Mirrorea の実現責務を切り分ける |
| D-019 | `all_of` aggregate evidence の最小要件 | `all_of` では、participating `place` の全員について、対応する local attempt / local prefix に対する success 側の local observation と、それを支える persistence evidence が outcome 判定に数えられて初めて full coverage が成立する。Mir-1 の event surface は aggregate attempt に対する full coverage 成立済みの success、またはそれ以前の明示的 failure outcome だけを意味語彙として残し、per-place evidence payload や具象表現は要求しない。audit surface は per-place の対応関係、counted local observation の有無、supporting persistence evidence が outcome 判定に数えられたかを追跡できなければならない。full coverage の不成立は success 不成立を意味するが、それだけで failure を確定しない | L1 | `all_of` の aggregate evidence を最小 coverage 条件に絞り、Mir-1 と Mirrorea の表現境界を明確にする |
| D-020 | `all_of` aggregate failure justification | `all_of` では full coverage 不成立だけでは aggregate failure を確定しない。aggregate failure event を正当化できるのは、required member の local attempt / local prefix に explicit local failure が立って同じ aggregate attempt の full coverage が不可能になった場合、または aggregate success より前にその aggregate attempt が failed outcome として明示的に閉じられた場合に限る。member-local failure event 自体は aggregate failure event の代用ではない。timeout-like budget、policy cancellation、retry exhaustion などの closure reason taxonomy は Mir-1 の独立語彙にせず、Mirrorea の operational policy / audit に残す | L1 | coverage 不足と failure 確定を分離しつつ、aggregate failure の十分条件だけを最小限で固定する |
| D-021 | `all_of` aggregate failure の audit trace 最小要件 | `all_of` の aggregate failure audit は、aggregate attempt、participating / required member 集合、各 required member と local attempt / local prefix の対応、justifying required member local failure または explicit failed closure の参照可能性、そして failure 観測時点または closure 時点の missing coverage 状態を説明できなければならない。event surface は aggregate failure outcome だけを保持し、`reason_ref` のような field 名、ID 形式、serialization、closure reason taxonomy は Mir-1 で固定せず、Mirrorea の audit representation に残す | L1 | 後から failure を説明できる最小監査情報だけを Mir-1 に残し、表現形式は Mirrorea に送る |
| D-022 | `all_of` missing coverage snapshot の基準時点と粒度 | `all_of` の missing coverage snapshot は、same aggregate attempt が aggregate failure outcome として明示的に観測された決定点を最小基準時点とする。explicit failed closure がそのまま failure outcome を明示する場合は closure 時点がこの基準時点になる。Mir-1 が要求する最小粒度は per-required-member 粒度であり、各 member について対応する local attempt / local prefix と結びつけた上で、counted success-side coverage あり、local failure により coverage 不可能、未充足の少なくとも三状態を区別できれば足りる。複数時点 snapshot、保存形式、ID 化、圧縮方式は Mir-1 で固定せず Mirrorea に残す | L1 | missing coverage の説明可能性だけを最小限で固定し、履歴管理や表現形式を Mirrorea に送る |

## Important note

A decision listed here is not immutable. It means only that the project currently prefers this direction at the stated level.
